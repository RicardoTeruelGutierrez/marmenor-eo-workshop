The three unlocks

  • Process API — send an evalscript, get the finished product (NDCI, true-colour, or raw bands) for any date.
  • Statistical API — a full AOI time series in one request, downloading no imagery at all.
  • Real numbers — pull the per-band reflectance and read the chlorophyll signature straight off the spectrum.

Register in ~2 min, store credentials as env vars, and the same cells run live.

module 3 / 04CDSE process + statistical API
§ 05 — Store it the right way

SQLite for queries,
Parquet for bulk.

CSVs become a swamp. A database gives types, primary keys (no duplicates), indices, and SQL. We use a pragmatic hybrid.

  • SQLite (one file, zero install): scenes, tidy observations, stations.
  • Parquet: the millions of per-pixel values.
  • Idempotent writes (INSERT OR REPLACE on a UNIQUE key): re-running never duplicates.
CREATE TABLE observations (
  obs_id   INTEGER PRIMARY KEY,
  date     TEXT, source TEXT,
  variable TEXT, zone   TEXT,
  value    REAL, units  TEXT,
  UNIQUE(date,source,variable,zone)
);
-- tidy, long format = easy SQL

§ 07 — Scale & takeaways

Make it grow on its own.

1Discover → subset → array/table is the same pattern for every source.
2Download only the window (COG) and summarise server-side (Statistical API) — move as few pixels as possible.
3Clean before you store; store tidy. Idempotent SQLite + Parquet scales to a scheduled, incremental job.
4The SQL transfers to PostGIS / DuckDB when you outgrow one file. Swap the AOI and the database builds itself.
